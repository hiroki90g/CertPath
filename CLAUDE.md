# IT資格取得支援アプリ「CertPath」要求・要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
**CertPath（サートパス）** - IT資格取得のためのコミュニティ型タスク管理アプリ

### 1.2 開発期間・体制
- **開発期間**: 1ヶ月
- **開発時間**: 平日1.5時間/日、土日6時間/日（総計約60時間）
- **開発者**: 1名
- **開発ツール**: Claude Code

### 1.3 技術スタック
- **フロントエンド**: React
- **バックエンド**: Firebase（Firestore、Authentication、Hosting）
- **デプロイメント**: Firebase Hosting

## 2. 要求定義（What & Who）

### 2.1 解決すべき課題
**メインペイン**: IT資格取得のためのタスク洗い出しに30分〜1時間かかり、モチベーション維持が困難

**具体的な課題**:
1. **タスク設計の非効率性**: 資格取得に必要なタスクを毎回ゼロから考える必要がある
2. **情報収集の手間**: ブログやYouTubeから成功事例を探し、手動でタスク化する作業
3. **孤独な学習**: 一人だと継続が困難、モチベーション維持が課題
4. **信頼性の不安**: 自分が立てた学習計画が適切かわからない

### 2.2 ターゲットユーザー
**プライマリターゲット**: IT系新人エンジニア（社会人2-3年目）

**詳細ペルソナ**:
- **年齢**: 24-27歳
- **職種**: エンジニア、IT関連職
- **現在のタスク管理**: スマホメモアプリ、Notion
- **学習目標**: IT系資格取得（ITパスポート、基本情報、応用情報、Salesforce、AWS等）
- **性格**: 少し面倒くさがり、一人だとサボりがち、人に見られると頑張れる
- **利用シーン**: 主に土日の自己研鑽時間

### 2.3 提供価値
1. **タスク設計の効率化**: 2-3分で学習計画を立てられる
2. **実証済みの学習パス**: 先人の成功事例をベースにした信頼性の高いタスクリスト
3. **コミュニティによるモチベーション維持**: 同じ資格を目指す仲間との進捗共有
4. **継続支援**: 孤独感の解消と相互励まし機能

## 3. 要件定義（How）

### 3.1 機能要件

#### 3.1.1 コア機能（必須実装）

**ユーザー認証機能**
- Googleアカウントでのログイン（Firebase Auth使用）
- プロフィール設定（ニックネーム、目標資格、学習開始日）

**資格テンプレート機能**
- IT資格のタスクリストテンプレート表示
- 対象資格：ITパスポート、基本情報技術者、応用情報技術者、Salesforce関連資格
- テンプレート詳細：タスク名、推定時間、優先度、参考リンク

**タスクリストカスタマイズ機能**
- テンプレートのコピー機能
- タスクの追加・削除・編集
- タスクの完了/未完了切り替え
- 進捗率の自動計算

**基本的な進捗管理機能**
- 個人ダッシュボード（進捗率、完了タスク数、残り日数）
- タスクの期限設定
- 完了したタスクの履歴表示

#### 3.1.2 差別化機能（コミュニティ機能）

**同じ資格チャレンジャー表示**
- 同じ資格を目指している他のユーザーの一覧表示
- ユーザーの進捗率とニックネームの表示
- プライバシー配慮：実名は非表示

**進捗共有機能**
- 自分の進捗を他のユーザーに公開（ON/OFF切り替え可能）
- 週次進捗レポートの自動生成
- 完了タスクの簡単な共有機能

**相互励まし機能**
- 他のユーザーの進捗に「いいね」「応援」を送信
- 簡単なコメント機能（「頑張って！」「お疲れ様！」等の定型文）
- 資格合格報告機能

### 3.2 非機能要件

#### 3.2.1 パフォーマンス要件
- ページロード時間：3秒以内
- リアルタイム更新：進捗変更が1秒以内に他ユーザーに反映

#### 3.2.2 ユーザビリティ要件
- モバイルファースト設計
- 直感的なUI/UX（説明不要で使える）
- PWA対応（ホーム画面追加可能）

#### 3.2.3 セキュリティ要件
- Firebase Security Rulesによるデータアクセス制御
- ユーザーデータの適切な匿名化
- 不適切なコンテンツのフィルタリング

#### 3.4.1 主要画面一覧

1. **ログイン画面**
   - Googleログインボタン

   - 各資格の概要と推定期間

3. **テンプレート詳細画面**
   - タスクリスト表示
   - 「このテンプレートを使う」ボタン

4. **マイプロジェクト画面**
   - 進捗ダッシュボード
   - タスク一覧（完了/未完了）
   - タスク編集機能

5. **コミュニティ画面**
   - 同じ資格チャレンジャー一覧
   - 励ましメッセージ履歴
   - いいね・応援機能

6. **プロフィール設定画面**
   - ニックネーム変更
   - 進捗公開設定
   - 目標変更

### 3.5 初期データ

#### 3.5.1 資格テンプレート（手動作成予定）

**ITパスポート**
- 基本的な学習タスク：約15個
- 推定期間：2-3ヶ月
- 参考書読了、過去問演習、模擬試験等

**基本情報技術者**
- 基本的な学習タスク：約20個
- 推定期間：3-4ヶ月
- 午前問題対策、午後問題対策、プログラミング言語選択等

**応用情報技術者**
- 基本的な学習タスク：約25個
- 推定期間：4-6ヶ月
- 高度な午後問題対策、論述対策等

**Salesforce関連資格**
- 基本的な学習タスク：約18個
- 推定期間：2-4ヶ月
- Trailhead学習、実践演習、模擬試験等

## 4. 成功指標（KPI）

### 4.1 ユーザビリティ指標
- **タスク設定完了時間**: 5分以内（目標：3分以内）
- **アプリ継続利用率**: 1週間後50%、1ヶ月後30%
- **タスク完了率**: プロジェクト開始ユーザーの50%が80%以上完了

### 4.2 コミュニティ機能指標
- **いいね/応援送信率**: アクティブユーザーの70%が週1回以上
- **進捗公開率**: 登録ユーザーの60%が進捗公開設定ON
- **コミュニティ画面アクセス率**: 週1回以上60%

### 4.3 ビジネス指標
- **ユーザー登録数**: 1ヶ月で50人（知人・SNS経由）
- **月間アクティブユーザー率**: 登録者の60%
- **NPS（Net Promoter Score）**: 50以上

## 5. リスクと対策

### 5.1 技術的リスク
**リスク**: Firebase初利用による開発遅延
**対策**: 事前に小さなサンプルアプリで動作確認、Claude Codeとの相談

**リスク**: リアルタイム機能の実装複雑性
**対策**: 最初は手動更新で実装、後からリアルタイム化

### 5.2 プロダクトリスク
**リスク**: ユーザー数不足でコミュニティ機能が機能しない
**対策**: 最初は知人に依頼してシードユーザーを確保

**リスク**: 不適切なコンテンツ投稿
**対策**: 定型文メッセージのみに制限、報告機能の実装

### 5.3 競合リスク
**リスク**: 既存大手サービスが類似機能をリリース
**対策**: コミュニティ機能による差別化、ニッチ市場での先行者利益確保
